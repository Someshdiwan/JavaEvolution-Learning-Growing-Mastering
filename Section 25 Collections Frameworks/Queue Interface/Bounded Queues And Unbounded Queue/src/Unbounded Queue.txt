An unbounded queue has no fixed capacity limit â€” it can grow dynamically
until available system memory is exhausted.

Producers adding elements never block, as the queue always accepts new items.

In Java:
- `LinkedBlockingQueue` â†’ optionally bounded, but unbounded by default (capacity = Integer.MAX_VALUE).
- `PriorityBlockingQueue` â†’ always unbounded, ordered by priority or natural order.
- `DelayQueue` â†’ always unbounded, elements become available only after their delay expires.

This design is useful when the number of elements is unpredictable and memory
resources are sufficient.

In short:
An unbounded queue favors flexibility and throughput by eliminating
capacity restrictions.

It is ideal for applications where queue size is expected to remain reasonable,
but it can be risky if producers outpace consumers â€” potentially causing
OutOfMemoryError due to uncontrolled growth.
